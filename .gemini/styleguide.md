# Gemini Style Guide (Router)

This file is intentionally short to avoid drift.

## Authority hierarchy (highest first)
1. `constitution/` — immutable principles
2. `contracts/` — agent rules + invariants + hard-stops
3. `requirements/` — domain specifications + checks
4. `docs/` — explanatory only

## Canonical entrypoints
- `constitution/constitution.md`
- `contracts/hard-stops.md`
- `contracts/invariants.md`
- `contracts/agents.md`
- `requirements/**/spec.md`
